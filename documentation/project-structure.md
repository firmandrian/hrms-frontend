## Project structure

This template was made with a well-defined directory structure.

```sh
documentation/
├── documentation.md # Full documentation for settings role, edit ui,etc
├── env-configuration.md # documentation for settings env by stage
├── project-structure.md #  documentation about project structure
├── readme.md # documentation for develop the project
├── tech-stack.md # documentation for tech stack used in the project
public/ # store all asset in this folder
src/ # To see a detailed explanation about src, you can look at the full documentation
styles/ # global styles
.env.development.local # env for dev
.env.staging.local # env for staging
.env.production.local # env for prod
```
